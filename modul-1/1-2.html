🏗️ HTML Structure

HTML is the skeleton of every webpage. Understanding how it is built — and which tags are dangerous — is the foundation of all web security work.

⏱️ ~25 minutes 📖 Reading + Interactive ⚡ Live code editor 🧠 4-question quiz

🎯 What you will learn

  • What HTML is and how a browser reads it
  • The structure of every HTML document — the skeleton
  • What tags, attributes, and elements are
  • Which HTML tags are most relevant to web security and why
  • How to read and write basic HTML in a live editor
📄 What is HTML?

HTML (HyperText Markup Language) is the language used to describe the structure and content of a webpage. It is not a programming language — it does not perform calculations or make decisions. It is a markup language, which means it uses special tags to label and organise content.

When your browser receives an HTTP response from a server, that response body is almost always an HTML document. The browser reads it top to bottom, interprets the tags, and renders the result visually on your screen.

ℹ️ Why HTML matters for cybersecurity
Every XSS attack works by injecting malicious HTML or JavaScript into a page. To find these injection points, to read source code during a pentest, and to understand why a payload works — you must be able to read and write HTML fluently. You are not learning this to build websites. You are learning it to break them.

🏷️ Tags, Elements, and Attributes

HTML is made of elements. Each element is defined by a pair of tags: an opening tag and a closing tag. Between them is the content.

🔐 Security Note — Attributes are injection points too
XSS payloads are commonly injected inside attributes. If a server puts your name into an HTML attribute without sanitizing it: <input value="YOUR_NAME"> — an attacker can break out with a quote and inject an event handler: <input value="" onmouseover="alert(1)">. You will practice this in Module 3.

🌳 HTML and the DOM Tree

When the browser reads HTML, it builds it into a tree structure in memory called the DOM (Document Object Model). Think of it like a family tree: <html> is the grandparent, <head> and <body> are its children, and every tag inside is a further descendant. Hover over the nodes below to explore.

📄document
└─<html>
├─<head>
└─<title>"My Page"
└─<body>
├─<h1>"Hello World"
└─<form>action="/submit"
└─<input>type="text"← injection point
🔐 Security Note — The DOM is the target
When JavaScript runs on a page, it interacts with the DOM — reading values, writing content, and responding to user actions. An XSS attack injects malicious JavaScript that manipulates the DOM: stealing cookies, creating fake login forms, or redirecting users to phishing pages.

⚠️ What just happened?
When you injected <b>bold</b>, the browser rendered it as bold text — it treated your input as real HTML, not as plain text. This is the core of HTML injection and XSS: the browser cannot tell the difference between HTML the developer wrote and HTML an attacker injected. In Module 3 you will see how this leads to full JavaScript execution.
